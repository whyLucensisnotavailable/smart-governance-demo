# 超大城市智慧治理 Demo

对应解决方案设计中的三条主线：

1. **城市治理大屏**：Leaflet 地图 + 城市实体（一物一码）+ 实时事件流
2. **暴雨内涝事件闭环**：信号回放 → 事件融合（时空窗口聚类）→ 多智能体研判（DeepSeek）→ 人工审核 → 跨部门工单 → 复盘指标
3. **政务服务智能助手**：RAG 问答，回答标注政策依据来源

## 快速启动

```bash
cd demo
pip install -r requirements.txt

# 可选：配置 DeepSeek Key（不配置则自动使用内置预设回答演示）
cp .env.example .env   # 然后编辑 .env 填入 DEEPSEEK_API_KEY

python backend/main.py
```

浏览器打开 **http://127.0.0.1:8000**

## 演示流程

1. 页面加载后，地图显示海淀区试点区域与约 20 个城市实体点位（点击可看一物一码档案）。
2. 点击左上角「**开始暴雨推演**」：8 条多源信号（气象/IoT/视频/市民/热线/交管）按时间轴回放。
3. 知春路下穿桥的 5 条信号被融合引擎聚合为 1 个「内涝风险事件」（苏州街 2 条信号保持"观察中"，演示去重与升级阈值）。
4. 事件生成后自动触发 4 个智能体（应急/交通/城管/公众服务）流式研判。
5. 点击「**人工审核 · 确认执行**」生成 5 张跨部门工单；点击「推进工单状态」走完 生成→签收→处置→复核→办结。
6. 全部办结后复盘面板显示全链路耗时指标。
7. 右侧政务问答可随时提问，例如：「暴雨天气积水导致车辆受损如何申请理赔」「积水多深会封路」。

## 关键说明

- **无 Key 兜底**：未配置 `DEEPSEEK_API_KEY` 时，研判与问答自动使用内置预设内容（与场景数据一致），全流程仍可完整演示；顶栏徽标会显示当前模式。
- **数据**：信号/实体/资源为模拟数据（采用真实路名与合理坐标）；政策语料节选整理自真实公开文件。
- **真实代码逻辑**：事件融合（时空窗口聚类+多渠道交叉印证）、RAG 检索（2-gram 重叠打分）、工单状态机均为真实运行的代码，不是预录动画。
- **AI 只做辅助决策**：工单只有在人工审核确认后才会生成，与方案中的可信边界设计一致。

```text
   ├─ app.js

```
